# L298N Motor Driver Validation Test

## Objective

The objective of this test is to verify the correct operation of the L298N motor driver and validate the GPIO control signals generated by the ESP32 before integrating the propulsion system into the complete autonomous vehicle.

---

# Engineering Rationale

During the early development phase, the L298N motor driver was used as a simple and reliable platform for validating motor control software before transitioning to the final high-current motor driver.

This approach allowed the software to be developed independently from the final power electronics, reducing debugging time and simplifying hardware testing.

---

# Hardware Configuration

## Components

- ESP32 DevKit V1
- L298N Motor Driver
- DC Motor (optional)
- External Battery (required for motor operation)

---

## ESP32 Wiring

| ESP32 | L298N |
|--------|--------|
| GPIO25 | IN1 |
| GPIO26 | IN2 |
| GPIO27 | ENA |
| GND | GND |

---

## Motor Wiring

| L298N | Motor |
|--------|--------|
| OUT1 | Motor Terminal A |
| OUT2 | Motor Terminal B |

---

## Power

During software testing, only the control signals were verified.

The DC motor requires an external battery and cannot be powered directly from the ESP32 USB connection.

---

# Software Description

The software executes the following sequence:

1. Initialize GPIO outputs.
2. Drive the motor forward.
3. Stop.
4. Drive the motor in reverse.
5. Stop.
6. Repeat continuously.

Serial messages are printed to assist with debugging.

---

# Test Procedure

1. Connect the ESP32 and L298N according to the wiring table.
2. Upload the validation program.
3. Open the Serial Monitor.
4. Observe the printed motor states.
5. If an external battery and motor are connected, verify correct forward and reverse rotation.

---

# Expected Serial Output

```text
L298N Motor Test Started

Forward
Stop
Reverse
Stop
```

---

# Validation Result

The ESP32 successfully generated the expected GPIO control signals.

The software correctly alternated between:

- Forward
- Stop
- Reverse
- Stop

This confirmed the functionality of the motor control logic.

---

# Engineering Notes

The L298N was used exclusively during the early software development stage.

The final competition robot will use a BTS7960 motor driver due to its:

- Higher current capability
- Lower power loss
- Better efficiency
- Improved thermal performance

The control logic developed during this test will be reused with only minor modifications in the final software architecture.
